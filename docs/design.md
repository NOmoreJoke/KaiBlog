# KaiBlog 前端设计文档 · v1.0

**受众**：视觉设计师 / 产品原型师 / 前端工程师
**目标**：提供能直接进入 Figma/Pencil 做高保真原型的视觉与交互规范
**版本基线**：对齐 `intent.md v3.0`（2026-04-23）
**参考对标**：Anthropic 官网（信息密度与节奏）、乔木博客（写作体验）、小宇宙 App 订阅页（封面横滚）

---

## 1. 设计原则（Design Principles）

### 1.1 四条底层纪律

1. **内容优先**：视觉永远为阅读服务，不为设计师的审美服务。任何装饰性元素若不帮助读者聚焦，一律砍掉。
2. **留白即节奏**：不用分割线和装饰，用间距建立层次。页面"空"比"满"更重要。
3. **单色克制**：主界面只用黑/白/灰三色。点缀色仅出现在状态标签、交互反馈、数据可视。
4. **移动优先 + 桌面友好**：任何页面先保证 375px 可用，再延展到 1440px。不做"PC 特效，手机丢内容"。

### 1.2 反模式（看到这些 PR 必 reject）

- ❌ 紫色渐变 / 毛玻璃 / 装饰性阴影
- ❌ 拟物化图标（建议用 Lucide 或自绘极简线性图标）
- ❌ 首屏视频自动播放、弹窗订阅、"浮动客服"
- ❌ "Awwwards" 那种视差和字符炫技
- ❌ 主体字小于 15px

## 2. 品牌气质（Brand Tone）

| 维度 | KaiBlog 是 | KaiBlog 不是 |
|---|---|---|
| 语气 | 克制、准确、内行 | 网红、激动、口号化 |
| 视觉 | 黑白印刷品、ID 卡质感 | 插画风 / 卡通 / 渐变 |

| 密度 | 内容满但不拥挤 | 稀疏留白堆砌高级感 |

一句话定位：**"一个黑白印刷品质感的数字杂志，作者每一篇都认真写"**。

## 3. 色彩系统（Color Tokens）

### 3.1 浅色模式（默认）

| Token | 值 | 用途 |
|---|---|---|
| `--ink-100` | `#0B0B0B` | 主要正文 |
| `--ink-80`  | `#1F1F1F` | 标题 |
| `--ink-60`  | `#4A4A4A` | 次要文字 |
| `--ink-40`  | `#8A8A8A` | 辅助文字、时间戳、图注 |
| `--ink-20`  | `#C8C8C8` | 描边、分隔（仅必要时） |
| `--paper`   | `#FFFFFF` | 底色 |
| `--paper-soft` | `#FAFAFA` | 卡片内嵌底（如音频卡片） |
| `--accent`  | `#0B0B0B` | 链接、按钮 = 与主文字同色（极简派） |
| `--accent-hover` | `#3A3A3A` | hover |
| `--success` | `#1F6E43` | 绿，仅用于状态 |
| `--warning` | `#8A5A00` | 琥珀 |
| `--danger`  | `#8A1F1F` | 暗红 |

### 3.2 暗色模式（二期做，但设计稿要预留）

| Token | 值 |
|---|---|
| `--ink-100` | `#F5F5F5` |
| `--paper`   | `#0B0B0B` |
| `--paper-soft` | `#171717` |

暗色不是简单反色：正文放弃纯白 `#FFF`，用 `#F5F5F5` 避免灼眼。

### 3.3 使用约束

- **一个页面最多出现 3 个灰度层级**（例如：ink-80 / ink-40 / ink-20）
- **accent 色不做彩色**：KaiBlog 没有蓝色链接，所有可点击元素通过下划线或细微 hover 区分
- 状态色（success/warning/danger）**仅用于小徽章/图标**，不能出现在 Hero 或大面积区块

## 4. 字体系统（Typography）

### 4.1 字体栈

| 用途 | 字体 |
|---|---|
| 中文正文 | `"PingFang SC", "HarmonyOS Sans SC", "Microsoft YaHei", system-ui, sans-serif` |
| 英文/数字 | `"Inter", "SF Pro Text", system-ui, sans-serif` |
| 标题可选加重 | 同正文，通过 weight 区分，不换字体家族 |
| 代码 | `"JetBrains Mono", "SF Mono", Menlo, monospace` |

**禁止**引入衬线字体、装饰字体、Web Font 体积 > 200KB 的选择。

### 4.2 字号层级（rem 基准 = 16px）

| 角色 | 桌面 | 移动 | weight | line-height |
|---|---|---|---|---|
| Display（首页 Hero） | 56px / 3.5rem | 36px | 600 | 1.15 |
| H1 (文章标题) | 36px / 2.25rem | 28px | 600 | 1.2 |
| H2 | 28px / 1.75rem | 22px | 600 | 1.25 |
| H3 | 22px / 1.375rem | 18px | 600 | 1.3 |
| Body-lg（文章正文） | 18px / 1.125rem | 17px | 400 | 1.7 |
| Body | 16px / 1rem | 16px | 400 | 1.6 |
| Small（时间、标签） | 14px / 0.875rem | 14px | 400 | 1.5 |
| Caption（图注） | 13px | 13px | 400 | 1.4 |

**Body-lg 1.7 倍行高是阅读长文章的关键**，不要为了"紧凑"压缩。

### 4.3 段落与排版

- **正文最大行宽**：`680px`（约 35-40 汉字）—— 超过就破坏阅读节奏
- **段落间距**：`1em`（不要用 `<br>`）
- **中英文混排**：自动加 `0.125em` 间距（可用 `auto-spacing` CSS 或 pangu.js，任选其一）
- **代码块内边距**：`20px 24px`；行高 `1.55`；底色 `--paper-soft`
- **引用块**：`border-left: 3px solid --ink-20`, `padding-left: 20px`, `color: --ink-60`

## 5. 间距与栅格（Spacing & Grid）

### 5.1 间距刻度（4px 基线）

```
4 · 8 · 12 · 16 · 24 · 32 · 48 · 64 · 96 · 128
```

**用法约束**：
- 组件内间距：4~16
- 卡片与卡片：24~32
- 区块与区块：48~64
- 首屏 Hero 下方空间：96
- 不允许出现 10px、15px 等非刻度值

### 5.2 容器宽度

| 场景 | 最大宽度 |
|---|---|
| 文章正文容器 | 680px |
| 首页内容容器（文章流） | 720px |
| 播客页（含横滚墙） | 960px |
| 关于我 Hero 区 | 960px |
| Admin 后台 | 1200px |
| 所有容器 | 左右边距最小 `24px`（移动端），`32px`（桌面） |

### 5.3 断点

```
sm: 640px
md: 768px
lg: 1024px
xl: 1280px
```

设计稿按 375 / 768 / 1280 三档出。

## 7. 核心组件视觉规格（Component Specs）

### 7.1 按钮

| 类型 | 样式 | 用途 |
|---|---|---|
| Primary | 黑底 `--ink-100` + 白字 + 圆角 6px + padding 12 20 | 主行为（发布、保存） |
| Secondary | 白底 + 黑字 + 1px `--ink-20` 描边 + 圆角 6px | 次行为（取消、返回） |
| Ghost | 无底无框 + ink-60 文字 | 第三级行为 |
| Destructive | 白底 + `--danger` 文字 + 1px `--danger` 描边 | 删除 |

按钮高度统一 **40px 桌面 / 44px 移动**（移动触控区更大）。

### 7.2 输入框 / 表单

- 高度 40px
- 底色 `--paper`
- 边框 1px `--ink-20`，focus 时 `--ink-80`
- 圆角 6px
- 内边距 12px
- Label 在上方，`small + ink-60`
- 错误态：边框变 `--danger`，下方 `small --danger` 错误提示

### 7.3 Tag / Chip / Pill

| 类型 | 样式 |
|---|---|
| Category Pill | 14px，padding 6 12，圆角 full，未选 `--ink-60` 文字无底，选中黑底白字 |
| Tag Chip | 13px，前缀 `#`，灰文本 `--ink-60`，hover 变 ink-80 |
| Status Badge | 11px，uppercase，带底色：`DRAFT` 灰、`PUBLIC` 绿、`LINK_ONLY` 琥珀 |

### 7.4 卡片（Card）

**铁律：卡片无阴影**。间距是唯一的分隔手段。

- 圆角 6px（与输入框统一）
- Hover 时：标题加下划线，其他不变
- 禁止出现 `box-shadow`、`border-radius: 12px` 以上、渐变底

### 7.5 Tiptap 编辑器

- 顶部 toolbar 高度 48px，8 个图标按钮 + 分隔线
- 图标：H1 / H2 / Bold / Italic / Link / Quote / Code / Image
- 编辑区最大宽度与文章正文一致（680px），但编辑器容器自身占满可用宽
- 选中文本不弹气泡菜单（契约 §4：简化版，不抄乔木 Notion 交互）
- 粘贴小宇宙链接 → 自动替换为 MediaEmbedCard 预览（带 "移除" 按钮）

### 7.6 MediaEmbedCard（内联版 / 卡片版）

**内联版**（文章内）：宽度占满文章容器（680px），与单集卡片一致。

**墙内版**（播客页顶部横滚）：80×80 仅封面，下方节目名。

### 7.7 SubscriptionShelf（横滚墙）

- 容器：`overflow-x: auto; scroll-snap-type: x mandatory;`
- 每个封面：`scroll-snap-align: start`
- 隐藏滚动条但保留功能
- 最多显示 30 个节目，之后加 "全部订阅 →" 链接跳到完整列表页

### 7.8 AI Assist 面板（后台）

右侧抽屉，320px 宽，从右滑入（200ms ease-out）。

## 8. 交互与动效（Motion）

### 8.1 动效原则

- **Duration 默认 150-200ms**，easing 默认 `cubic-bezier(0.4, 0, 0.2, 1)`（ease-out）
- **页面切换不做 transition**（浪费时间，用户只想快速看下一页）
- **任何 hover**：只改颜色/下划线，不做 transform / scale
- **反馈动画最多 250ms**：提交成功的对勾、复制成功的 toast
- **禁用动画**：长度超过 300ms 的任何动画一律砍

### 8.2 滚动
- 原生行为，不做 smooth scroll polyfill
- 横滚容器**不做**滚动指示器动画

---

## 9. 无障碍（Accessibility）

底线要求（不做满分，但不能漏）：

- 所有图片必须有 `alt`
- 颜色对比度：正文 ≥ 4.5:1、大标题 ≥ 3:1（已由 ink-80/paper 组合达成）
- 键盘焦点可见：`outline: 2px solid --ink-80; outline-offset: 2px`
- 按钮和链接 `aria-label` 在图标场景必加
- 视频/音频嵌入 iframe 必带 `title`

## 10. 响应式策略

### 10.1 断点行为

| 断点 | 导航 | 内容容器 | 播客封面墙 | 作品集网格 |
|---|---|---|---|---|
| < 640px | 4 Tab 等宽横排 | 100% - 48px | 72×72, 显示 4.5 个 | 1 列 |
| 640-1024px | 同上 | 680px 居中 | 80×80, 显示 6 个 | 2 列 |
| ≥ 1024px | 顶部右对齐 | 720px 居中 | 80×80, 显示 8+ | 3 列 |

### 10.2 字号自适应

不搞 `clamp()` 魔法，只在 `< 640px` 媒体查询里把 Display 从 56px 降到 36px、H1 从 36px 降到 28px，其他保持。

## 11. 图标（Iconography）

- **Lucide**（推荐，线性风格与黑白气质一致）或 **Heroicons** outline 版
- 图标统一 stroke-width 1.5px
- 尺寸只允许：16 / 20 / 24 / 28
- 禁用 Emoji 作为正式图标（仅允许在社交栏和技能徽章等"装饰性情境"）

## 13. 关键取舍（Design Decisions Log）

| 决策 | 选择 | 理由 |
|---|---|---|
| 是否做圆形头像 | ❌ 用 8px 圆角方形 | 圆形太"社交产品"，与印刷气质不符 |
| 是否有彩色链接 | ❌ 链接与正文同色 + 下划线 | 避免蓝色污染整体黑白调性 |
| 是否做毛玻璃导航 | ❌ 实色白底 | 毛玻璃是 2020 的 iOS 审美，已过期 |
| 是否做站内音频播放器 | ❌ | 聚合模式定位，跳原平台 |
| 是否支持用户评论 | ❌ 首期 | 不做 |
| 是否做深色模式 | ⚠ 二期 | 设计稿预留 token，首期不实现 |
| 是否做文章目录 TOC | ❌ 首期 | 680px 宽度 + 阅读时长提示已足够 |
| Logo 形态 | ✅ NOmoreJoke 几何字标系统 | 以左上角开口方框中的 `NO` 为主识别，`moreJoke` 为展开层 |
| 首页顶部 | ✅ 无 Hero，直接内容流 | 与乔木博客一致的"打开就是内容"节奏 |
| 作品集封面 | ✅ 统一抽象图案背景 + 标题 + 简介 | 视觉一致性优先于信息密度；降低每次新作品的配图成本 |
| 关于我 Bio | ✅ Tiptap 富文本（支持 H2 / 列表 / 链接 / 引用） | 保留叙事弹性，但默认不用 H1（H1 是页面标题专属） |

## 14. Logo 的具体取舍（补充 §13）

**方向定了：NOmoreJoke 极简几何字标系统**。

Logo 不再使用单汉字字标。主识别资产是左侧接近正方形的开口框 `NO` 模块，右侧 `moreJoke` 是内容展开层。详见 `docs/NOmoreJoke_logo_style_document_zh.md`；首版 SVG 资产位于 `docs/logo/`。

**主标结构**：

- 左侧：左上角开口方框 + 几何化 `NO`
- 右侧：保持原始大小写的 `moreJoke`
- 下划线：只作用于 `moreJoke`，不贯穿完整 Logo

**硬性规则**：

- `NO` 是主识别核心，不能与 `moreJoke` 同权
- 外框必须接近正方形，开口必须固定在左上角
- `moreJoke` 不改成全大写、全小写或 `NoMoreJoke`
- 主体必须在纯黑白状态下成立
- 禁止卡通、涂鸦、赛博霓虹、渐变、阴影、伪 3D

**使用规范**：

- 主标：`docs/logo/nomorejoke-logo.svg`，用于页眉、首页、页脚、品牌展示
- 反白主标：`docs/logo/nomorejoke-logo-reverse.svg`，用于深色背景
- 折叠版：`docs/logo/nomorejoke-mark.svg`，用于移动端、小导航、头像位
- 微型图标：`docs/logo/nomorejoke-favicon.svg`，用于 favicon 和标签页
- 主题色版：`docs/logo/nomorejoke-logo-accent.svg`，仅用于大尺寸品牌展示
- 主标最小宽度 160px；折叠版最小尺寸 24×24；favicon 目标尺寸 16×16 / 32×32
- 小尺寸版本禁用主题色，优先保证左上角开口和 `NO` 可识别
